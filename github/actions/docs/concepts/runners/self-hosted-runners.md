# Собственные исполнители (Self-hosted runners)

Вы можете размещать собственных **исполнителей (runners)** и настраивать среду, используемую для выполнения **заданий (jobs)** в ваших **сценариях автоматизации Гитхаб Экшнс (GitHub Actions workflows)**.

**Собственный исполнитель (self-hosted runner)** — это система, которую вы развертываете и которой управляете для выполнения заданий из **Гитхаб Экшнс (GitHub Actions)** на **Гитхабе (GitHub)**.

Собственные исполнители:
* Предоставляют больше контроля над **оборудованием (hardware)**, **операционной системой (operating system)** и **программными инструментами (software tools)**, чем **исполнители, размещенные на Гитхабе (GitHub-hosted runners)**. Помните, что вы несете ответственность за обновление операционной системы и всего остального программного обеспечения.
* Позволяют использовать машины и сервисы, которые ваша компания уже содержит и оплачивает.
* Бесплатны для использования с **Гитхаб Экшнс (GitHub Actions)**, однако вы несете расходы на обслуживание ваших машин для исполнителей.
* Позволяют создавать индивидуальные конфигурации оборудования, отвечающие вашим потребностям в вычислительной мощности или памяти для выполнения крупных заданий, а также устанавливать программное обеспечение, доступное в вашей локальной сети.
* Получают автоматические обновления только для самого **приложения собственного исполнителя (self-hosted runner application)**, хотя вы можете отключить автоматическое обновление исполнителя.
* Не требуют наличия «чистого» экземпляра (**clean instance**) для каждого выполнения задания.
* Могут быть физическими, виртуальными, находиться в **контейнере (container)**, развертываться **локально (on-premises)** или в **облаке (cloud)**.

Вы можете использовать собственных исполнителей на любом уровне **иерархии управления (management hierarchy)**. **Исполнители уровня репозитория (repository-level runners)** выделены для одного репозитория, в то время как **исполнители уровня организации (organization-level runners)** могут обрабатывать задания для нескольких репозиториев в организации. Владельцы организации могут выбирать, каким репозиториям разрешено создавать собственные исполнители уровня репозитория. См. раздел «Отключение или ограничение Гитхаб Экшнс (GitHub Actions) для вашей организации». Наконец, **исполнители уровня предприятия (enterprise-level runners)** могут быть назначены нескольким организациям в корпоративном аккаунте.
